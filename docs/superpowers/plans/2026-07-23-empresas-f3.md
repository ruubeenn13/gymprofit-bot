# Empresas · Fase 3 (Economía) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Encender la economía de las empresas: corte del curro al bote + bonus de ingresos por nivel, subir nivel gastando del bote, nómina diaria por rango, y ascensos de tier patrocinados por el bote.

**Architecture:** Migración mínima V29 (columna `dato`) → repo (bote/nivel + dato) → nómina (función pura + job) → corte/bonus en `/trabajo currar` → `/empresa mejorar` + info → ascenso patrocinado (refactor de `ascender` + gobernanza F2). Constantes de balance en la spec.

**Tech Stack:** Java 21, JDA 5, Flyway/MySQL, JUnit 5 + Mockito, Testcontainers, jobs `ScheduledExecutor`, i18n, `EmbedFactory`.

**Spec:** `docs/superpowers/specs/2026-07-23-empresas-f3-design.md`. **Build de referencia:** 496 tests.

**Patrones a imitar:** `EmpresaRepositorio` (JDBC + Testcontainers), `EjercicioDiaJob` (job diario 03:00 Europe/Madrid + Clock), `TrabajoService.ascender`/`currar` (economía atómica), `EmpresaGestionService` (gobernanza F2), `EmpresaComando`/`EmpresaBotonesListener` (subcomandos + botones/autocompletado).

**Constantes (en un sitio, p. ej. `EmpresaEconomia` o el service):** `CORTE_EMPRESA=0.10`, `BONUS_POR_NIVEL=0.02`, `NIVEL_MAX=10`, `COSTE_NIVEL(n)=50_000*n`, `FRACCION_NOMINA=0.20`, peso rango = `ordinal()+1`.

```markdown
## ADR-018 — economía de empresas

**Estado:** aceptada e implementada (Fase 3).

**Contexto.** El bote y el nivel de las empresas (columnas desde V27) estaban inertes; faltaba el
motor económico y el sentido de pertenecer.

**Decisión.** Un **10 % del curro** de cada miembro va al **bote**; el **nivel** (subido gastando del
bote, `/empresa mejorar`) da **+2 %/nivel de ingresos** a todos (tope +20 %); una **nómina diaria**
reparte el 20 % del bote por rango; y el bote **patrocina ascensos de tier** de los miembros
(`/empresa ascender`, vía la gobernanza de F2). Se refactoriza `TrabajoService.ascender` en
validación/aplicación reutilizables para el patrocinio sin duplicar reglas.

**Consecuencias.** Migración mínima V29 (columna `dato` en las propuestas para el puesto del ascenso);
`bote`/`nivel` ya existían. `TrabajoService.currar` depende ahora de `EmpresaRepositorio`. Nuevo
`NominaEmpresasJob` (03:00 Europe/Madrid).
```
